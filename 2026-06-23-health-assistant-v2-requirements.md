## 3. 目标用户 👥

| 画像 | 痛点 | 我们的解药 |
|---|---|---|
| 🧑‍💻 减脂期上班族 | 知道要管住嘴但不知道今天吃啥 | 早上 30 秒拿到今日 3 餐 + 1 练 |
| 💪 增肌爱好者 | 不会自己配高蛋白餐 | 餐餐标注克数 + 蛋白克数 |
| 👵 慢病中老年 | 医生建议笼统、小红书不知对错 | 内置 DASH / MIND 等方案 |
| 🆕 坚持不下来的人 | 工具太冷、缺少反馈 | 心情打卡 + 历史曲线 + 鼓励文案 |

**不服务**：16 岁以下、孕产妇、临床糖尿病/肾病/肝病人群（页面底部固定医学免责声明）。

## 4. 核心场景 🌅

### 场景 1：早晨打开（典型）☀️
小李 7:30 醒来 → 打开「健康助手」 → 看到昨天已经为今天生成的 **2 张大卡片**：
- 🥗 **饮食卡片**（早/午/晚 3 餐，配 AI 写实摄影风插图，豆包生成）
- 🏃 **运动卡片**（类型 + 时长 + 强度，配 AI 动态插画，豆包生成）

### 场景 2：晚上打卡 🌙
小李 22:00 回到页面 → 快速点几下记录今天：
- 🍱 实际吃了什么（早/午/晚/加餐，可文字可搜索）
- 🏋️ 实际做了什么运动（类型 + 时长）
- ⚖️ **今天体重** 67.0 kg
- 😊 **今天心情** 😊 + 一句话日记

### 场景 3：回看趋势 📈
小李点「历史」 → 看到：
- 📅 最近 7 天的饮食 / 运动 / 心情日历
- ⚖️ 体重曲线（绿涨红跌的折线图）
- 🔥 连续打卡天数 + 小勋章
- 💬 AI 写的一句周复盘鼓励

### 场景 4：方案不兼容 ⚠️
小张选了「16+8 断食 + 增肌」→ 弹非阻塞提示「这两个搭配不完美匹配，要换一个吗？」

### 场景 5：AI 故障 🛟
豆包 / LLM 挂掉 → 顶部黄条提示"AI 暂不可用，已为你展示基础建议" + 卡片降级为静态插画 + 数字照常。

## 6. 数据模型 🗄️

> Supabase Postgres（行级安全 RLS 开启，用户只看自己数据）

```
profiles
  id (uuid, FK auth.users)
  gender, age, height_cm, weight_kg, goal, activity
  diet_plan, notes
  updated_at

daily_advices
  id, user_id, date (YYYY-MM-DD)
  meals (jsonb), exercise (jsonb), summary
  image_diet_url, image_exercise_url
  source: 'llm' | 'fallback'

daily_logs
  id, user_id, date
  meals_actual (jsonb)
  exercise_actual (jsonb)
  weight_kg (numeric)
  mood (1..5), mood_note (text)
  created_at

moods (可选字典)
  id, emoji, label
```

> 索引：`(user_id, date)` 复合索引；`daily_logs` 用 upsert 保证一天一条。

## 7. 技术架构 🛠️

| 层 | 选型 |
|---|---|
| 前端框架 | **Next.js 15（App Router）+ TypeScript** |
| 样式 | Tailwind CSS + shadcn/ui |
| 表单 | react-hook-form + zod |
| 数据库 | **Supabase（Postgres + Auth + Storage）** |
| 鉴权 | Supabase Auth（邮箱 + 魔法链接） |
| LLM | 兼容 OpenAI 协议的国内模型（豆包为主，DeepSeek 备用） |
| 文生图 | **豆包 Seedream API** |
| 部署 | **Vercel**（Next.js 原生） |
| 代码托管 | **GitHub** https://github.com/zhouxiaoyingai/myhealth.git |
| 监控 | Vercel Analytics + Sentry（可选） |
| 测试 | Vitest + Playwright（E2E） |

```
Browser (PC/Mobile)
  └─ Next.js App Router (SSR + RSC)
       ├─ /api/advise   ──► 豆包 LLM
       ├─ /api/image    ──► 豆包 Seedream
       └─ Supabase Client ──► Postgres (RLS)
Vercel (Edge/Serverless)
GitHub (CI: PR preview + Vercel auto deploy)
```

## 9. 非功能需求 ⚙️

- **性能**：首屏 LCP < 2.5s；`/api/advise` P95 < 8s；前端打包 < 250KB gz
- **兼容**：Chrome / Safari / Edge / Firefox 最近 2 版；iOS 15+ / Android 10+；320–2560px
- **可用性**：键盘可达；颜色不单通道；中文字号 ≥ 14px
- **安全**：HTTPS only；API Key 不入 git；CSP 设置；Supabase RLS 必开
- **隐私**：未登录数据仅本地；登录后云端但 RLS 隔离；可一键导出 / 删除
